# DOC_RULES.md

各ドキュメントの運用ルール。

---

## ■ ファイル責務と更新タイミング

| ファイル | 書くべき内容 | 更新タイミング |
|---|---|---|
| `README.md` | 概要・起動方法・ファイルマップ・操作一覧 | ファイル構成変更時・操作変更時 |
| `spec.md` | ゲーム仕様（機能・挙動・ルール）。数値は balance.md に委譲 | 仕様変更・機能追加時 |
| `architecture.md` | システム構造・コード設計・座標系・関数の責務 | コード構造変更時 |
| `balance.md` | 数値パラメータのみ（HP・速度・ダメージ等） | パラメータ調整時 |
| `data.md` | アイテム・敵・レシピのデータ定義。実装状況を必ず記載 | データ追加・実装完了時 |
| `HANDOVER.md` | 引き継ぎガイド・コアコンセプト・既知の未実装 | 重要設計変更時・未実装解消時 |
| `tasks.md` | 開発タスクの進捗管理。未実装・拡張候補・技術負債を含む | タスク着手時・完了時・技術負債発見時 |
| `issues.md` | バグのみ（症状・原因・対象を記載）。技術負債・未実装はtasks.mdへ | バグ発見時・解消時 |

---

## ■ 書いてはいけないこと

| ファイル | 禁止内容 |
|---|---|
| `balance.md` | 仕様の説明文・ルール記述（→ spec.md へ） |
| `data.md` | 実装されていないデータを「実装済み」と記載 |
| `spec.md` | 未確認の仕様を確定仕様として記載（不明点は ※要確認 と明示） |
| `tasks.md` | バグ報告（→ issues.md へ） |
| `issues.md` | 新機能のタスク・技術負債・拡張候補（→ tasks.md へ） |
| すべて | 推測による仕様補完・コードと一致しない記述 |

---

## ■ 実装とドキュメントの同期ルール

1. **機能を実装したら**：tasks.md の該当タスクを `[x]` に変更
2. **バグを発見したら**：issues.md に追記（症状・原因・対象ファイルを記載）
3. **バグを修正したら**：issues.md から削除または「解決済み」に変更
4. **パラメータを変更したら**：balance.md の数値を更新
5. **data.md のデータを実装したら**：実装状況欄を ✅ に変更
6. **設計を変更したら**：architecture.md を更新してから実装する

---

## ■ 禁止事項

- 情報の削除（統合・移動はOK）
- 推測による仕様の補完
- コードと一致しない情報の放置
- ファイルの責務を無視した記述
